✅ Success Criteria

To successfully meet the best practices for HTML, CSS, and JavaScript, you need to select at least 12 out of the 15 checkboxes below.
If all 15 are checked, even better! A message will appear below the checklist to let you know if you’ve met the success criteria.

1

Use Semantic HTML: Improves accessibility, SEO, and code readability.

My website uses <header>, <nav>, <main>, etc.

2

Use Proper Meta Tags: Helps SEO and better preview on social media.

My site includes <meta name="description"> and viewport tags.

3

Use Alt Attributes for Images: Improves accessibility and SEO.

My images include descriptive alt text.

4

Label Your Forms: Improves screen reader support and usability.

My form inputs use associated <label for> elements.

5

Validate HTML Code: Ensures cleaner, error-free markup.

I checked my code with W3C Validator.

6

Use External CSS: Keeps HTML clean and code reusable.

I use <link rel="stylesheet"> instead of inline styles.

7

Make It Responsive: Supports mobile and tablet devices.

I use @media queries for layout responsiveness.

8

Use CSS Variables or Preprocessors: Improves maintainability and consistency.

I use :root variables or tools like SASS.

9

Minify and Optimize CSS: Improves performance by reducing file size.

I use minified styles.css for production.

10

Use Consistent Naming: Makes the codebase scalable and readable.

I follow naming like block__element--modifier.

11

Use External JS: Keeps HTML cleaner and separates logic.

I use <script src="main.js"> instead of inline JS.

12

Use let and const Instead of var: Provides better scoping and avoids bugs.

I declare variables using let and const.

13

Avoid Excess DOM Manipulation: Improves performance and user experience.

I batch DOM changes or use document fragments.

14

Use Event Delegation: Efficient event handling for dynamic elements.

I use event delegation with event.target.

15

Use try-catch and Error Handling: Makes your app more reliable and user-friendly.

I handle errors using try-catch blocks and fallback logic.
